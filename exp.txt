# AI-Powered Brain Tumor Radiotherapy Planning Pipeline
*End-to-End Multimodal System: Segmentation → Clinical Report → RT Contouring*  
**Dataset:** BraTS2020 | **Compute:** Kaggle 2× Tesla T4 | **Year:** 2025–2026

---

## 1. Project Overview

Built a fully automated, end-to-end AI pipeline for brain tumor radiotherapy planning on the BraTS2020 dataset. One of the first systems to integrate deep learning segmentation, multimodal large language model reporting, retrieval-augmented generation (RAG) clinical guidance, and automated GTV/CTV/PTV contouring into a single unified framework — running on dual-GPU commodity hardware (2× Tesla T4, 29.2GB total VRAM).

**Four-stage architecture:**

1. **3D UNet** — segments whole tumor (WT), tumor core (TC), and enhancing tumor (ET) from multi-modal MRI (T1, T1CE, T2, FLAIR)
2. **MedGemma 1.5** — generates structured clinical radiology reports from segmentation outputs and T1-CE images
3. **RAG Clinical Guidance Agent** — Llama-3 + FAISS hybrid retrieval grounded in ESTRO-EANO 2023 published guidelines
4. **GTV/CTV/PTV Contouring** — GPU-accelerated 3D morphological operations implementing two validated clinical protocols with brain-boundary constraints

---

## 2. What I Built

### 2.1 Tumor Segmentation — 3D UNet
- Custom 3D UNet with MONAI sliding-window inference over full 240×240×155 NIfTI volumes
- Multi-class output: NCR (label 1), Edema (label 2), Enhancing Tumor (label 3)
- Pre-trained checkpoint (epoch 80, mean Dice 0.8136) deployed on `cuda:1`
- **Dice WT=0.9288, TC=0.8748, ET=0.8876** — competitive with published BraTS SOTA

### 2.2 Clinical Report Generation — MedGemma 1.5
- Integrated Google's MedGemma 1.5 (4B-IT), a medical vision-language model built on Gemma 3
- Correct Gemma 3 chat template: `apply_chat_template()` with structured messages list and PIL image — not raw `<image>` token strings
- Runs in `bfloat16` on `cuda:0`; input_len slicing to decode only newly generated tokens
- GPU memory management: `torch.cuda.synchronize()` + `empty_cache()` + `max_new_tokens=256` before generation

### 2.3 Clinical Guidance Agent — RAG Pipeline
- Llama-3 4-bit quantized (BitsAndBytes QLoRA) as generator LLM on `cuda:0`
- Hybrid retrieval: FAISS dense vector search (BGE embeddings) + BM25 sparse retrieval via `EnsembleRetriever`
- Cross-encoder reranker for top-k document selection before LLM context injection
- Knowledge base: 370 pages of ESTRO-EANO 2023 + RTOG protocol guidelines via `PyPDFLoader` + `RecursiveCharacterTextSplitter`
- LangChain `RetrievalQA` chain with custom `PromptTemplate` for structured RT recommendations

### 2.4 GTV/CTV/PTV Contouring — Dual Protocol
- **ESTRO-EANO 2023:** GTV (ET+NCR) → 15mm CTV → 3mm PTV (single phase, 60Gy/30fx + temozolomide)
- **RTOG dual-phase:** GTV+edema → 20mm CTV46 (46Gy) + GTV → 20mm CTV60 (60Gy) → 3mm PTVs
- GPU-accelerated 3D dilation using `torch.nn.functional.max_pool3d` on `cuda:1` (~50× faster than scipy CPU)
- Brain boundary masking via T1CE intensity threshold + `scipy.ndimage.binary_fill_holes` to clip contours at skull
- Clinical visualization: contour outlines via `skimage.measure.find_contours` overlaid on T1-CE MRI with combined overlay panel
